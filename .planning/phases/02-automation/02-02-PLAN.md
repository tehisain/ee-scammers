Write deploy.yml with this exact structure:

```yaml
name: Deploy site

on:
  push:
    branches: [main]
    paths-ignore:
      - 'src/data/**'   # skip deploys triggered by data-only commits

permissions:
  contents: read

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      # Phase 3 inserts here:
      #   - uses: actions/setup-node@v4
      #     with:
      #       node-version: 'lts/*'
      #       cache: 'npm'
      #   - run: npm ci
      #   - run: npm run build
      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v4
        with:
          path: .   # Phase 3 replaces with: dist/

  deploy:
    needs: build
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

Key implementation notes:
- `paths-ignore: ['src/data/**']` — nightly scraper commits update src/data/chart-data.json; this filter prevents unnecessary deploys from those commits
- Workflow-level `permissions: contents: read` is minimal; deploy job overrides with pages+id-token
- `upload-pages-artifact@v4` and `deploy-pages@v4` — MUST be v4. v3 was deprecated January 30, 2025 and will fail
- `path: .` is a temporary skeleton value. Phase 3 replaces with `dist/` after adding the Astro build step
- `environment: github-pages` is required for the OIDC token exchange that authenticates the deploy
- Comments in the build job mark the exact Phase 3 insertion point — executor must not remove them
- The deploy job structure is final — Phase 3 does NOT modify it
