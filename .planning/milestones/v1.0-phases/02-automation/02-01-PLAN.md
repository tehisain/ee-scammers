  <action>
Create .github/workflows/ directory and write scrape.yml with this exact structure:

```yaml
name: Scrape data

on:
  schedule:
    - cron: '0 3 * * *'   # 03:00 UTC nightly
  workflow_dispatch:        # allow manual trigger

jobs:
  scrape:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 'lts/*'
          cache: 'npm'

      - run: npm ci

      - name: Run scrapers
        run: npm run scrape

      - name: Commit updated data if changed
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add src/data/chart-data.json
          git diff --cached --quiet || git commit -m "chore: update chart data [skip ci]"
          git push

  keepalive:
    if: github.event_name == 'schedule'
    runs-on: ubuntu-latest
    permissions:
      actions: write
    steps:
      - uses: liskin/gh-workflow-keepalive@v1
```

Key implementation notes:
- Cron `0 3 * * *` = 03:00 UTC every day (user-specified; accept minor queue drift per research)
- `workflow_dispatch` enables manual test trigger without waiting for cron
- `contents: write` on scrape job is required — git push will 403 without it
- Commit message includes `[skip ci]` so this data commit does NOT trigger deploy.yml
- `git diff --cached --quiet || git commit` — exits 0 (no diff = skip commit) or 1 (has diff = commit). Push always runs; harmless if nothing to push
- keepalive job uses `liskin/gh-workflow-keepalive@v1` — NOT `gautamkrishnar/keepalive-workflow` (that repo is ToS-suspended by GitHub Staff as of early 2025; using it causes "Access to this repository has been disabled" error at action download time)
- keepalive `if: github.event_name == 'schedule'` — runs only on cron, not on workflow_dispatch or push
- `actions: write` permission on keepalive job is required for GitHub API re-enable call
- Use `node-version: 'lts/*'` (not a hardcoded version) so workflow tracks LTS automatically
- `cache: 'npm'` speeds up installs on repeat runs
